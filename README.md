## About

`dev-sec-cheatsheets` is the official repository maintained by **DigitalBulwark**. It serves as a unified knowledge base for advanced command-line techniques, DevSecOps configurations, and vulnerability assessment methodologies.

Unlike standard listicles, this repository focuses on **practical, high-value snippets** utilized in daily professional environments. We enforce strict formatting and contribution guidelines to maintain the quality and reliability of the knowledge base.

## Categories

The repository is structured into domain-specific documentation:

- **[Network Security & Reconnaissance](./docs/network-security/)**: Advanced network mapping, Nmap scripting engine triggers, and firewall evasion techniques.
- **[DevOps & Infrastructure](./docs/devops/)**: Docker hardening, CI/CD pipeline security, and infrastructure-as-code snippets.
- **[Version Control](./docs/version-control/)**: Advanced Git recovery, bisecting techniques, and secure branching models.
- **[Web Penetration Testing](./docs/web-pentest/)**: Payload catalogs, WAF bypass methodologies, and automated fuzzing dictionaries. *(Coming Soon)*

## Contributing

We welcome contributions from the community, ranging from minor typo fixes to the addition of entirely new domain categories.

To maintain the architectural integrity of this repository, all contributors **must** strictly adhere to our [Contributing Guidelines](./CONTRIBUTING.md).

### Quick Start for Contributors:
1. Fork the repository.
2. Create your feature branch (`git checkout -b feature/AddNewSnippet`).
3. Commit your changes using Conventional Commits (`git commit -m 'docs: add new docker hardening rules'`).
4. Push to the branch (`git push origin feature/AddNewSnippet`).
5. Open a Pull Request for review by the maintainers.
