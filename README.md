## About

`dev-sec-cheatsheets` is an open-source initiative designed to be the ultimate quick-reference guide for IT professionals. 

We realized that developers and security engineers often waste valuable time searching for specific, advanced terminal commands or configuration snippets during production incidents. This repository serves as a centralized, community-verified knowledge base to solve that problem. 

**Our focus is on quality over quantity.** We only accept practical, situational snippets that are ready to be used in real-world environments.

## Categories

The repository is structured into domain-specific documentation:

- **[Network Security](./docs/network-security/)**: Advanced network mapping and service enumeration techniques.
- **[Web Security](./docs/web-pentest/)**: Common payload matrices and web vulnerability testing tools.
- **[DevOps & Cloud](./docs/devops/)**: Docker container hardening, Kubernetes configurations, and automated deployments.
- **[Automation](./docs/automation/)**: Python execution scripts and daily Bash automation.
- **[Version Control](./docs/version-control/)**: Advanced Git recovery techniques and incident management.

## Contributing

We welcome contributions from everyone in the community—whether you are fixing a typo or adding an entirely new category. 

To maintain the high standard of this knowledge base, all contributors are required to follow our structured review process. Please read our [Contributing Guidelines](./CONTRIBUTING.md) before submitting a Pull Request.

### Quick Start:
1. **Fork** the repository and create your feature branch.
2. Ensure your snippets are tested and accurate.
3. **Commit** using Conventional Commits.
4. **Push** and Open a Pull Request using our automated PR template.

Your code will be reviewed by community maintainers. Thank you for helping build a better resource for everyone.
